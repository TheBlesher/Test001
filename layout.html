Layout

It is always a good practice to look at the layout and try to understand the boundaries of the cell and how the part should flow through it.


The boundaries are usually set by fences and light curtains.

In this cell we have Material Handler robots, Welding robots and MIG Welding robots. MH robots have a special end of arm tooling (EOAT) designed specifically for handling applications, in other words these are robots capable of taking (picking) and placing (dropping) parts from one place to another. When the robot only handles the picking and dropping it is a normal MH robot, but if it is involved in a process application, like picking a part, taking it to a sealer pedestal application (without releasing the part) and then dropping it in another place, then this is a coop MH. Although there are different joining applications, like laser weld, arc weld, spot weld, rivets, etc. In this example we only have spot and arc weld robots. Spot welding is a resistance welding process that uses large electrical current to join two or more sheets of metal in a single location, and an arc welding robot uses a process which applies intense heat to metal at a joint, causing the metal to melt and intermix.

The first thing we see is station CN150 surrounded by 4 robots, just by looking at it we can assume adjacent cell’s CN140R04 drops a part into CN150, then CN150R01 and CN150R02 do a welding process in said station and finally CN150R04 picks the part from it. You can see where each robot’s path is by looking at the path outline which is in red and purple. It shows the edges of where each robot will reach on its path.


If CN150R04 picked, that means it should drop into CN160 since that is the other fixture its outline covers, then the 3 welders should do their process and finally CN160R04 picks the part. As you can see, the naming also helps us get an idea of how the robots work, the previous station’s last robot drops into the current station, Robots CN160 R01/R02/R03 weld and the last robot of the stations takes the part away.


Following the same train of thought, we can easily tell how the robots will interact in station CN170.


Sometimes you might thing, “is that robot really going to reach that fixture?” We can think that about CN170R04, but by looking at its path outline, which is officially named “Robot’s Work Envelop”, we can tell all the places it will interact with and make sure someone has already made sure it can be reached.

Sometimes things can be a little bit more complicated. After CN170R04 picks one part from CN170, by looking at its path envelop we can be sure it will drop into CN180, that is because its path envelop is not considering the nearest fixture CN200T01, but then, what happens next? According to CN180R01’s path outline this robot can reach CN200T01, CN200T02, CN190, CN180, CA010T01 and CA010T02. When unsure about something related with the part flow, and something you need to check just to be completely sure of what you are doing, is to go through the Control Lineups. In there you will be able to tell how everything should flow, what part present will be turned on or off, what clamp will be activated or deactivated, etc.


The Control lineups will tell us what, when and where everything is happening.

  • 1) The resource name tells us the device that is doing or where something is happening. This is the “where”. The device and description further increase the information we get.
  • 2) Function description gives us detailed information on what is happening, this is the “what”.
  • 3) And the start time, duration and end time give us precise information of when the action is happening.


I colored steps 2.A and 6.A because they are happening at the same time (they have the same start time), but not just those two, 10.A and 14.A also happen at the same time. This means that robot CN180R01 will initiate, go to pounce, then from pounce it can go to pick from either CN180, CN190, CN200T01 or CN200T02. After picking it will continue on step 18.A only if it picked from CN180 and then once again we have multiple places where it can drop, 23.A, 27.A, 31.A, 35.A and 39.A start at the same time, meaning CN180R01 can drop to either CA010T01, CA010T02, CN200T01, CN200T02 or CN190.

This might not make much sense at first glance, but basically this is what happens:

CN170R04 drops on CN180, CN180 is an inspection fixture where the part will be checked every set number of cycles, if the part has issues in its weld the operator needs to manually fix the part. Then CN180R01 picks the part and takes it to an RVC camera to be inspected for further weld problems, if the part is determined to be okay it can be drop onto CA010T01 or CA010T02 (whichever is available), if the part has any kind of suspects it will be drop either on CN190 (if it is ready to receive the part), or CN200T01/CN200T02 (these are option specific, so it will drop depending on the type of part).

Now let us look into CN160’s control lineups.


In a fixture’s control lineup, you will notice it includes the actions happening on it (part presents and clamps being activated or deactivated), the robot that picks and the one that drops on it, and also the robots that have a process on it.

On step 1 CN150R04 drops the part on the station (we have CN150_R04 as the resource). Then on step 2.A PPX01, PPX02 and PPE03 get activated, in the comments we can see that both PPX01 and PPX02 are common parts and PPE03 will be active when REINF-R/SEAT OBD ATTACH INR LH - BT1CC VALUE is in the station. In this cell we have the following styles (to know the styles your cell has you need to check the NAX sequence build plan, it is located in the next folder: K:\Operations\App_Dev\Controls\SW\GM\Orion\BT1xx\11_Documents\Style_NAX Sequence Build Plan).


BT1CC Value is a reference to a part with value options. This time we have Value and Retail options, value should have Option[0].0(CFF), Option[0].1(ESG) and Option[0].3(QK2) and retail only Option[0].2(Base EndGate). This also is related with stations CN200, because CN200T01 is used for value and CN200T02 for retail.

After the part presents turn on, the robot opens its Gripper 1 (do not worry about this, in GM the robot handles all its clamps and part presents so we won’t simulate them). After that it goes to part present check, clear of drop, clear of station and goes home. Then the fixture closes clamp 1.


On step 5.A it should say “Close Pin Clamp 1”, but still, we can see it is closing both clamp 1 and pin clamp 1 at the same time. We are “Ret” C05A and C06A (meaning we are retracting both pin clamps), and we are also “Ext” C02AL and C02AR (meaning we are extending both clamps). After this the welders initiate and start their welding process. Then the clamps are opened and finally CN160R04 picks the part (it says CN160R03 but it is obviously wrong, because this is a welder robot and cannot pick).
