Start creating a WinMOD file

• Create a new folder and gather all the important files (WinMOD template, Libraries, Lineups, Layout, Prints, WinMOD DesignTool, Generators and MPHWLib).


• Add the driver to your WinMOD file.


•Add the most recent MPHWLib.


•Work on the WinMOD-DesignTool.


o Add new Devices and their Device Type, by looking into the Hardwire Prints.
o Start by adding the devices that are at Cell level.
>Don't add Operator Display Panels
>Example: Add the PLC tag "AL003G1" > Device Type "Gate Box"
>Some Naming:

Remote HMI > Display Box
Entrance Gate > Gate Box
RACK INTERFACE PANEL > ECS-5093
OPERATOR FIXTURE PANEL > ECS-5525

>Don't forget to look at the bottom of the document for I/O Blocks and other devices that are connected at Cell level.
>When encountering a Safety Block ad "su_" before its name.
o Then add the devices in each station.
>Make sure you use the right item by looking into the BOM (Bill of materials).
oOnce done with the Design Tool press Generate IO and you will get two “.tab” files

o Import the TAGS generated in "Output_Tags".
>Right click on Driver "B110 Allen Bradley Ethernet/IP Logix5"
>Click on "Import tag list..." > Look for your file "Output_Tags.tab"

Make sure to select: Tabulator > None > Windows > at table header > at table end > Hit Next.
Check mark all but the last one. > Click Finish.


o Import operands generated in "Output_OprndAloc".
>Right click on Driver > Operand Allocation > Import from File > Select the file that was created "Output_OprndAloc.tab".
>Tabulator > None > Windows > at table header > at table end > column names in 1st line checked.
>All Signal Types Checked > None of the Naming Convention Checked
>Match the Internal with the Assigned (you can remove and assign them)

o Start adding Devices in WinMOD, you can add "Custom" files from Libraries.
>Start in the same order you have them in the WinMOD Design Tool.
>In 00-Nav you can add a capture of the whole cell and create shortcuts for the other windows.

>In 01-Main you will locate the Safety connections and Material flow connections.

>In 02-Disp... add Gate Boxes, Display Boxes (HMIs), and Cell level components.
>In 02-Disp... also add the safety blocks and cell level blocks.

>In 03-Oper... add Operator Panels.
>In 04-Robots... add the robot controllers and any component related to them, like cameras.

>Next add the stations one by one.

When adding a fixture add Part Presents, Valves, and their Blocks.
When adding Racks or other components add the macro and make sure the connections are ok by following the prints.
Copy the "XX###X##.sim" example file, rename it and add it from WinMOD.
There are some macros you can use and modify later.
Mostly you will add Valves (not each clamp) and Part Presents (PPX).
Add AIR switches if necessary.
Add its BKs and PMs.


>Macros like Conveyors, SISO carts, Hokutos, etc. Will need some modifications, in most cases connections for part presents, but it is always a good practice to compare the item with the prints to verify its functionality.

o Add Connections, look at the Hardwire Prints to know how to make the connections.
>Go through the Hardwire Prints, start with the cell level stuff and then go station by station, make sure all connections are made.
>Start by adding the Safety Modules under the Main Screen

>Make sure the panels are the right ones, if not, compare the ECS and modify accordingly.

>Don't forget to look at everything for any missing stuff, like a camera or something like that, you can always look into the ethernet devices in order to see all the individual items that will be on the cell.
>Remember to drive every AuxPwr... in all SBKs and BKs with Outputs.

>It is always good practice to comprehend your cell by looking at the layout and picturing how everything should work.
>The robot macro comes only with 2 picks and drops so, when adding a Robot make sure it goes to the right segments and completes all its processes, you can sim it on itself and modify it accordingly.
>You can simulate your robots to be sure they will work as intended during SIM and avoid wasting time.

o Working on Material Flow
>We need to simulate a part going through all the cell from start to finish, this means we need to make it "appear" or "disappear" in every fixture by means of a robot Picking or Dropping, a Hokuto PRB transferring the part, etc.
>Before you begin working on the material flow it is recommended to look at the layout and comprehend how everything is located inside the cell.
>Once you have a basic understanding of the layout go through the control lineups and start adding conditions to turn on/off the part presents on every station and device (SISO, CV, etc.).
