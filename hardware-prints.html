Hardware prints

Accessing files.


The two sections you will need to get are the controls lineups and the Hardware PDF’s.
The hardware PDF’s folder sometimes contains subfolders and sometimes, such as in this example files case, contains a lose collection of pdf’s and excel files.


Copy both of these folders into a project folder in your local drive before editing or opening them. Once copied you can remove the excel files from the Hardware PDF folder. This is not necessary, but it may make it easier to find what you are looking for. Next find the cell overview. It will be named with the same number as the cell name and often looks a bit different from the rest of the PDF names.


Our cell print is this file.


If you are unsure open the pdf. If it is the cell overview print the first page should look similar to the image below.


Many of the rest of the PDF’s will be station prints. They will look similar to the image below.


You can also see the stations listed in the small dotted boxes on the first page of the cell prints. It may be helpful to rename the PDF files so that you can find them easier. There is also robot prints in this folder, their name will be differentiated due to containing “R##” such as “CN150R04”.


Major parts of print file

All print files have sheet numbers, they can be found on the bottom right side of each page.

These section and sheet numbers may be referenced in other parts of the print. Such as when safety block inputs are connected to multiple Estops. Often when refenced there is a link to the page but not always.


Cell overview:


The sections of the cell print that we will focus on are the components and station list, the ethernet allocation table, and safety blocks.
The components and station list is on the first page of the cell overview. In the large solid box contains most of the major devices in the cell. Below the box are smaller boxes which contain the name and type of stations within the cell.
The Ethernet Allocation table is towards the bottom of the PDF, it is just before the safety blocks connections.

In this section you can see all the devices that have an IP address. It often gives devices such as safety blocks that are not listed in the components list but the two lists often overlap. It may also be missing some devices such as robots. It is important to check both lists.
The section directly below the Private Ethernet Address Allocation Table is where safety blocks and J-box connections can be found. These connections will be important when creating a file.


Station prints:

On the first page of station prints you can see the name of the station, the name of the cell it is in, and the type of station it is. For example, station 160 is a respot fixture.

Other important sections of the station print are the block connections, and any pneumatic manifolds. On the left is the block connections and on the right is the pneumatic manifold.


The Hardwire prints are divided into Cell level and Station level prints, at cell level you will find devices like Robots, HMIs (Human Machine Interfaces), operator run stands, entrance gate boxes, rack interfaces, etc. All the devices that have any kind of safety interaction (Safety Blocks, devices with an E-Stop, etc.)


We will start identifying the different items at cell level and decide how to add them to our simulation file.

It is important to identify the right element to add it if it already exists, or to modify or create it if it is needed. ECS refers to Electrical Cabinet Standard, an ECS is a standard industrial control panel designed by GM, being standard means they are always the same, so you can reference its design specifications for a better understanding.

You can find GM ECS’s at: K:\Operations\App_Dev\Controls\SW\GM\_Standards\02_Design Specs\GM ECS\ECS Files

By looking into ECS-4035 “Operator Guard Box” you can see it has a keyed selector with Teach/Tool Minor, Auto and a muting lamp.


But if we look at page 41 of our cell level prints, we can see it being connected to CN150SBK02 alongside the Lockable disable box and the operator run stand.


This means we will add them individually and make the connections directly to a Safety Block, hence, we don’t need to add them as an item in the WinMOD Design Tool.

We do not simulate Operator display panels so you can skip them.

Entrance gate control panels, Remote HMIs, Robots and J-Boxes for Racks are elements we simulate, make sure the right ECS is being used when adding it to your simulation and Design Tool. By the way, you don’t need to add the J-Box because it is already included inside the SISO macro, just make sure the connections are properly made using ECS-5093 as reference.

We only simulate signal connections, so it is not necessary to add any PDP or PLC in a box elements. That being said we can skip all the Power Distribution Panel connections, WPDP connections, Hot power connections and control connections.

While we do not simulate ethernet connections it is important to have some understanding of them. It helps us to identify all the components of the cell and it comes in handy while doing the ENET section of the VC checklist.


  • CN150ES1 refers to cell Ethernet switch 1, it is a 24-port ethernet switch and its IP is 120.15.209.15.
  • This means CN150SBK02 is connected directly to CN150ES1’s port 23.
  • CN150BK11 is connected to CN150SBK02 and not directly to the ethernet switch. While doing the ENET section of the checklist you will need to pay special attention to this, because you will need to make sure the elements are connected to the right element in there.

After the ethernet connections we have the ground connections and then the E-Stop daisy chained robot connections. You will reference these connections later when doing the connections to the SBKs.

Then we have the safety blocks, station’s CN150 safety block 01, 02 and 03. On page 54 we have the BOM and on item number 14800 we can see these safety blocks are 12in/2out Molex safety blocks.


  • 1) On top of each block, you will find its associated name: CN150SBK01, this tells us this is the first Safety Block belonging to cell CN150. You will also find how many ports it has and its IP.
  • 2) In the inputs part of the safety block, we will connect all the devices that send a safety signal to the PLC. Being a safety block means the inputs will be redundant, having two channels per input. On input 0 we will connect operator’s light screen CN180LC1 channel 0 and on input 1 we will connect channel 1. On inputs 8 and 9 we will connect CN180LC2 and on 10 and 11 we will have CN180LC3. This means we need to add 3 light curtains to our simulation. The robot E-stops will be connected the same way, each channel will go to a different input, the only difference is that they will be daisy-chained, meaning that all robots channel 1 will be connected to the same input and the other channel of every robot in the array will go to the next input. Reference sheet CM1 to see each robot array.
  • 3) On the output side of the safety block, we have all the devices we will be powering from this safety block. On outputs 0 and 1 we have the Auxiliary Power of station’s CN150 Pneumatic Manifold 1 “CN150PM1”.


Safety blocks and normal blocks are similar to this one, just make sure to add the elements and make the appropriate connections. In some special scenarios you might need to add buttons or switches and manually make the connections if needed.

This is one of those special scenarios where you need to add individual items into WinMOD and make sure they are connected properly to the safety blocks.

Once you have finished adding all the elements from the cell level print continue with every station. There are prints for Fixture stations and for special stations like conveyors, SISOs, Hokutos, etc.

Let’s take as an example Station CN150B01 – Respot Fixture


Looking at the prints we can tell it has two clamps, two pin clamps and 3-part presents, all connected to the same I/O Block.


This is a 16 input I/O block, the connections are similar to the safety block, the only difference being we won’t have two channels per input.


We have two clamps “Pin clamp 1” and “Clamp 1”. The individual valves such as “Pin clamp C05A” and “Pin Clamp C06A” are connected to the same clamp in the manifold meaning they will move together. This diagram is of the pneumatic manifold, and it shows the control of air flow. It shows the initial position, what type of clamps are used, and how many clamps there are. This is PM1 you can see the name of the PM underneath the clamp name and its positions.


As you can see, in PM1 we only have 2 valves, this means we will drive two valves from that manifold.
Each grouping has one valve. Since the valve is what changes the air flow it makes them move together. The valve name and the clamps it includes are boxed in green. For example Pin Clamp 1 includes C06A and C05A.


The most common valve is the 2-way one, but sometimes you can find 1 and 3-way ones. Make sure to add the right one when doing your simulation file. Both of the valves in our example are 2 way valves.
Each valve type has a distinct purpose and layout. The one way valve was one position where is is pushing air in and one where it is allowing air out. The arrows within the valve show the direction of air flow allowed, the lines will line up and connected to the clamps. The flat line on top of the incoming line means that the line is blocked off. For example, the middle position on the 3 way valve blocks air flow in either direction. The 1 and 3 way valves have tension applied such as a spring force, this is shown by the zig zag line on the side which it puts pressure. The 3 way valve applies equal pressure to either side meaning when not given an input it will sit in the middle slot. The 1 way valve has a spring on one side meaning when no input is applied it will return to its starting position. The 2 way valve has no springs. These differences help to determine which type of valve is being used.


On WinMOD side (and in the real world), this means we will have 4 different connections to the I/O block per set of clamps in this case (because we have different clamp states) and only 2 connections per valve on the manifold (because we are only driving one valve). The different clamp states are at home and at work. Each individual clamp has an at home and at work state. So for the pin clamp we have at home and at work twice since one correlates to C05A and the other to C06A. You can see in the image for BK01 a couple images above which specific input corresponds with C05A and C06A.

There are also stations that consist of a specific device like a rack, conveyor, SISO, etc. Usually, we already have an item designed to simulate this kind of device, but still you will need to make sure the connections are done properly because often they are unique between cells and stations. Often the mapping of the part presents, sensors, and clamps change. Always go into the station layout and look at the block connections and compare them to the device connections on the provided item. Sometimes an entire block will be added, removed, or renamed. This may mean you need to manually generate the internal block tags as the ones made from the device tag generation will be inaccurate.

This time we have such stations, CN190T01 is a SISO cart and CA010T01 and CA010T02 are conveyors. The naming for these conveyors is different because they belong to the next station, still we handle the loading of parts on them. They will have the name of the cell they end in. There can be load only, unload only, and full conveyors. Full conveyors are less common. You can often see which it is by the prints, what parts are called, naming in the prints, as well as looking at where the conveyor is in the line layouts. They are usually fairly easy to see as large green blocks in or between cells. The conveyor name and type is based off which part the current cell controls, an unload conveyor only unloads parts/receives them from another cell from before it, a load conveyor only loads parts to send to the next cell, and a full conveyor both loads and unloads parts typically used to move parts across the cell to reset the position of the line.


These are Accumulating conveyors with electric rotary stops. Instead of using clamps to hold the pallets, this conveyor uses 2 and 3 position rotary stops to hold and release the pallets. On one side we have the pre-load and load stops, and on the other we have the pre-unload and unload stops, and in the middle, we have mid stop 1 and mid stop 2. Each stop will hold or release the pallet depending on what’s needed. Only the load and unload have 3 positions, using this third position to lock the pallet in place to pick/drop the part. Depending on the size a conveyor can have up to 8 mid stops.

The connections are distributed and by looking at the prints you can tell what is handled by the load and what by the unload. We have 1 IDC1, 2 SBKs and 5 BKs.

By looking at both SBK11 and SBK12, we can tell that IDC1 and BK04 are connected to SBK11, and BK02 and BK05 are connected to SBK12.

By looking at BK02 we can tell this block handles the load side, and then on BK01’s input 14 we have the Load stop in position, meaning this block is handled by the load side. On the other side, we can see BK03 and BK04 have inputs and outputs related to the unload side. These blocks were not included in this document but can be found in the conveyor prints.

This means that SBK11, BK03, BK04 and IDC1 are handled by the unload side and SBK12, BK01, BK02 and BK05 are handled by the load side.

Knowing this we can go into our WinMOD macro and make sure it has the right connections, most of the time the stops are in the same place, and you might only need to modify the part present mapping.

For the SISO this time we didn’t get a print for the element itself and only for the “cart’s table”, meaning this is a standard SISO and we only need to pay attention to the table/fixture connections. PPE09 and PP10 are option specific, meaning they will turn on based on certain options related with the part being build. For this option you need to add a special bit that you will receive from the plc each time a part has those conditions. For convenience you can use a User Defined Bit belonging to any robot (because they are already mapped into the PLC and also in WinMOD).
