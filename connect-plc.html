Connect to a PLC

•Once the PLC is connected setup the IP address
•Connect to a PLC
o You may need to create a new connection between your PC and VM first
>Edit > Virtual Network Preferences > Change Settings > Add Network > Bridged to your Ethernet adapter on the Host.
>VM > Settings > Select the right Network Adapter > Custom and select your newly created Network.
o First connect your PC to the PLC
o Add your PC's ethernet adapter and VM ethernet adapter to the same network as the PLC.
>Go into its settings and configure the IPV4 IP and Subnet mask.
>The IP address should be in the same network as the PLC.

i.e. PLC= 120.29.124.131 then for your main PC use 120.29.124.150
This way your PC is using slot number 151 on network "120.29.124", as per subnet mask "255.255.255.0 > X.X.X is the network, and the last digit are the devices"
Do the same for the VM, use something like 120.29.124.151, make sure this one is not being used by another device.

>In the Subnet mask use the right one depending on the devices being used

i.e. in this example and most of the time you will use 255.255.255.0 as the subnet mask because it allows up to 253 devices

o Add the device using RsLinx.
>Add an ethernet/ip device and select the one with the ip address you configured for your PC or VM.

In this example I connected mine in the VM to 120.29.124.151

o Connect WinMOD to the PLC
>Right click the driver inside WinMOD > go to Target PLC > write down the PLC IP and select the right slot for the PLC > Test connection, it should say ok.
>How to know PLC's IP: Look into DCDL or just look at PLC's IP and use it.
o Read Tag Types
>If all your tags are named the same as in the PLC after reading the tag types, you should have no issues.
>If at reading the tag types, you find any issues check with your PLC guy to make sure the right naming or size of the tags.
